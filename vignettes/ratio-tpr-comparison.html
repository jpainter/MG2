Ratio Datasets and the Paired-Sum Rule: TPR as a Case Study

Overview

MG2 can compute derived ratio datasets — such as test positivity rate (TPR) — by combining separate numerator and denominator data elements downloaded from DHIS2. This vignette uses bundled Lao PDR malaria surveillance data to:

  1. Show how to build a TPR combined dataset from facility-level data using build_combined_dataset().
  2. Explain the validity rules MG2 applies when numerator or denominator values are missing for a given facility-month.
  3. Quantify how many facility-months fall into each category.
  4. Compare the MG2-computed TPR against the official DHIS2 indicator value for the same metric.

A companion section at the end shows how to download DHIS2 indicator values directly via the analytics API.

Data source: Lao PDR DHIS2 national HMIS (public demo instance, https://demos.dhis2.org/hmis), malaria programme. All demo data is bundled with MG2 and does not require a DHIS2 connection to run this vignette.

2 What is TPR, and why compute it in MG2?

Test positivity rate = confirmed positive tests ÷ total tests performed. In Lao PDR’s DHIS2 instance, the official indicator MAL - Malaria test positivity rate (Micr+RDT) (ID: acDxsQjnEof) computes:

\[ \text{TPR} = \frac{ \text{Microscopy positive} + \text{RDT positive} }{ \text{Suspects tested (microscopy)} + \text{Suspects tested (RDT)} } \]

MG2’s Combine module reconstructs this ratio from facility-level age/sex-disaggregated data, which enables:

  • Per-facility TPR for outlier detection and champion-facility analysis
  • Transparent documentation of exactly which data elements and cleaning levels were used
  • Age- and sex-stratified positivity rates

The trade-off is that facility-level data is noisier and requires careful handling of partial reporting — which is the focus of this vignette.

Each row is one facility × month. The numerator and denominator columns hold the summed counts across all selected age/sex categories for that facility-month. The value column holds numerator / denominator (or NA when the denominator is zero or missing).

4 Handling missing values in ratio datasets

4.1 The problem

In routine health facility data, it is common for either the numerator (positives) or denominator (total tests) to be absent for a given facility-month. How these cases are handled materially changes the aggregated ratio.

Consider three facility-months:

Facility Microscopy pos. RDT pos. Suspects (micro) Suspects (RDT) Numerator Denominator
A 5 7 30 50 12 80
B NA 3 NA 40 3 40
C NA NA NA 35 NA 35
D 4 2 NA NA 6 NA
  • Facility A: complete data. TPR = 12/80 = 15%.
  • Facility B: uses only RDT (no microscopy). With na.rm = TRUE, the age/sex sums collapse to RDT only. TPR = 3/40 = 7.5%. This is valid — the facility tested only by RDT.
  • Facility C: reports test counts but no positive cases. Leaving the numerator blank to save data-entry time is common practice when zero positives were found. TPR should be treated as 0/35 = 0%, not excluded.
  • Facility D: reports positives but no test-count denominator. This is an invalid record — you cannot compute a meaningful TPR without knowing how many tests were performed. Positives without a denominator would artificially inflate the aggregated numerator.

4.2 MG2 validity rules

MG2 applies three rules to each facility-month before aggregating:

Rule Condition Action Rationale
1. Missing numerator numerator is NA, denominator > 0 Set numerator = 0 Zero positives is plausible; blank entry is common practice
2. Orphaned numerator numerator is present, denominator is NA or 0 Exclude both Cannot compute TPR without total tests
3. Impossible ratio numerator > denominator Exclude both More positives than tests is not possible

Rule 1 is the key asymmetry: a missing denominator is an error, but a missing numerator is not — it most likely means zero.

4.3 How many facility-months fall into each category?

fac <- tpr[effectiveLeaf == TRUE]

fac[, .(
  total_rows             = .N,
  # Rule 1: missing numerator → will be imputed to 0
  missing_num_valid_den  = sum(is.na(numerator) & !is.na(denominator) & denominator > 0),
  # Both present and valid
  both_valid             = sum(!is.na(numerator) & !is.na(denominator) &
                               denominator > 0   & numerator <= denominator),
  # Rule 2: orphaned numerator → excluded
  orphaned_numerator     = sum(!is.na(numerator) & (is.na(denominator) | denominator == 0)),
  # Rule 3: impossible ratio → excluded
  impossible_ratio       = sum(!is.na(numerator) & !is.na(denominator) &
                               numerator > denominator),
  # Both missing
  both_missing           = sum(is.na(numerator) & is.na(denominator))
)]
#>    total_rows missing_num_valid_den both_valid orphaned_numerator
#>         <int>                 <int>      <int>              <int>
#> 1:      28678                 18843       8931                 69
#>    impossible_ratio both_missing
#>               <int>        <int>
#> 1:              594            0

The large missing_num_valid_den count — facilities that submitted test-count data but no positivity results — is treated as zero positives under Rule 1. This is by far the most common pattern, and the right call: a blank numerator in a country reporting low-transmission malaria is almost certainly zero, not unknown.

The orphaned_numerator count (facilities that reported positives without a denominator) is small but consequential: including these without a denominator would add positives to the national numerator sum with nothing in the denominator, inflating national TPR. MG2 excludes them.

The impossible_ratio cases (numerator > denominator) were flagged by the over_max = 1 setting during the combine step and excluded.

4.4 Effect on national TPR

To see the impact, compare national TPR computed three ways:

fac_2023 <- fac[as.Date(yearmonth(as.character(Month))) >= as.Date("2023-01-01")]

national <- fac_2023[, .(
  # MG2 rule: impute missing numerator = 0, exclude orphaned/impossible
  tpr_mg2      = sum(fifelse(is.na(numerator) & !is.na(denominator) & denominator > 0, 0, numerator),
                     na.rm = TRUE) /
                 sum(denominator[!is.na(denominator) & denominator > 0 &
                                 (is.na(numerator) | numerator <= denominator)],
                     na.rm = TRUE),
  # Old rule: require both non-NA (excludes the zero-positive facilities)
  tpr_paired   = sum(numerator[!is.na(numerator) & !is.na(denominator) &
                               denominator > 0 & numerator <= denominator],
                     na.rm = TRUE) /
                 sum(denominator[!is.na(numerator) & !is.na(denominator) &
                                 denominator > 0 & numerator <= denominator],
                     na.rm = TRUE),
  # Naive: na.rm on both independently (includes orphaned denominators)
  tpr_naive    = sum(numerator, na.rm = TRUE) / sum(denominator, na.rm = TRUE),
  n_mg2        = sum(!is.na(denominator) & denominator > 0 &
                     (is.na(numerator) | numerator <= denominator))
), by = Month]

The MG2 rule gives the most interpretable result:

  • It is lower than the old paired rule because it now includes the large denominator mass from zero-positive facilities, correctly pushing TPR down when testing is high and positives are low.
  • It is higher than the naive rule in periods where orphaned denominators (denominator with no corresponding numerator) were common — because MG2 excludes those orphaned denominators from the sum.

The differences trace to several sources:

Source Effect Explanation
Facility coverage MG2 lower counts MG2 uses only facilities in the downloaded dataset; DHIS2 includes all reporting facilities
Zero-imputation MG2 lower TPR when zero-positive facilities common MG2 imputes missing numerator = 0; DHIS2 may not, depending on how data are stored
Outlier cleaning MG2 may exclude extreme values mad10 removes extreme numerator/denominator values before computing the ratio
Category scope Depends on DHIS2 configuration DHIS2 indicator sums all category option combos; MG2 sums only the selected age/sex categories. If separate “total” rows exist alongside disaggregated rows in DHIS2, the indicator double-counts them; MG2 does not

Summary

Step What MG2 does Why it matters
Combine step Sums age/sex categories per facility-month Collapses disaggregation to one row per facility × month
Missing numerator Imputed to 0 when denominator is valid Zero positives is plausible; blank entry is common practice
Orphaned numerator Excluded (numerator present, denominator missing/zero) Cannot compute meaningful TPR; would inflate national numerator
Impossible ratio Excluded (numerator > denominator) Not biologically possible
Reporting tab Computes weighted average = Σnum / Σden Correct for ratio data; do not average facility-level TPR values
Champion analysis Based on denominator reporting presence A facility “shows up” for TPR if it reports total tests, even with zero positives
